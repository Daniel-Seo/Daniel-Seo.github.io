Partition을 고려해 볼만한 상황

  • 너무 많은 데이터
  • RAM이 부족한 경우
  • 많이 쌓여진 데이터
  • 꾸준히 증가하는 데이터

Partition 이란 논리적으로 나누어진 테이블

  • 테이블을 분할하여 생성할 필요가 없다
  • 파일을 이동하며 많은 양의 데이터를 옮길 필요가 없다

Partition을 하는 이유

  • 단일 insert문과  select문을 빠르게 처리하기 위해
  • range select 를 빠르게 처리하기 위해
  • 다른 경로사이의 데이터를 나누는 것을 도와주기 위해
  • 데이터 쌓기를 효율적으로 하기 위해
  • 많은 양의 데이터를 즉시 지우기 위해

Partition 종류

  • RANGE
  • LIST
  • HASH
  • KEY

Partition 제약 사항들

  • 정수형 컬럼만 파티션이 가능하다
  • 최대 파티션의 개수는 1024개이다
  • 만약 Unique Key 혹은 PK가 포함되었다면 파티션 컬럼은 해당 키의 부분이어야 한다
  • Foreign key는 불가능
  • Fulltext 혹은 GIS 는 불가능
